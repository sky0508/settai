## 画面一覧（対応モック）

### 1. お店を探す（ホーム）
`desktop-home-search.png` / `mobile-home-search.png`
- 条件フォーム。先頭に **来客フィールド**：既存ゲストを選択するとチップ表示＋直下に**カルテ要約**（好み / NG / 反省メモ）を出す。リンク「別のゲストに変更」「＋新しいゲストを追加」。
- フィールド: 取引先企業 / エリア / 予算 / 人数 / 個室希望トグル。
- 金の「探す」ボタン。
- ※来客は **任意**（未指定でも会社ルール＋条件で探せる。初回は「＋新規追加」へ誘導）← モックの「必須」表示は誤り、要修正。

### 2. 探索結果
`mobile-results.png`（デスクトップはホーム下に同レイアウトで展開）
- 店カードのランキング。各カード: 写真 / 店名 / ジャンル / 価格帯 / **ステータスバッジ（OK/Caution/NG）** / **フィット度 xx/100** / **理由一言**。
- 理由はゲスト履歴・反省を参照（例: 「前回から2ヶ月空き・重複回避」「前回の反省を反映すると騒がしい可能性」）。
- **NG も消さず最下部に理由付きで残す**（新人が学ぶ・最終判断は人間）。

### 3. 店舗詳細
`desktop-store-detail.png`
- 写真 / 店名 / ジャンル / 価格帯 / OK バッジ＋フィット度。
- **スコア内訳**: 4軸スコアバー = 格式 / 予算適合 / シーン適合 / 特別感（各 0–10）。
- **競合ドリンク確認**: 系列バッジ（例「サッポロビール系」）＋注意文（「キリンのお客様には注意」）＋**確信度（確実 / 推定 / 不明）**。
- **おすすめ理由**: その場で使える自然文（旬・個室・相手の好みに触れる）。
- 情報行: 個室有無 / 駅徒歩 / 要予約。CTA「予約メモを作成する」。

### 4. ゲストカルテ
`desktop-guest-karte.png` / `mobile-guest-karte.png`
- プロフィール: 氏名・役職・会社 / 好みタグ / NG タグ / 予算目安 / 接待頻度。
- CTA「このゲストに合う店を探す」（→ ホームに来客プリセットで遷移）。
- **接待履歴タイムライン**: 各回に 店名 / 年月 / ★評価 / **良かった点** / **反省メモ**。末尾に「＋接待履歴を追加」。

### 5. 接待を記録する
`desktop-record-form.png` / `mobile-record-form.png`（モバイルは夜に片手入力想定）
- 入力: お店 / 日付 / ゲスト / 同席者。
- **総合評価**（★5段階）。
- **良かった点**（チップ複数選択: 料理 / 個室 / サービス / 話題 / 店の格）。
- **反省点・次への申し送り**（自由記述, 〜200字）。
- **業務成果**（セグメント単一選択: 商談前進 / 関係深化 / 次アポ獲得 / 特になし）。
- CTA「記録する」。※ カルテ内「＋接待履歴を追加」からも遷移。

### 6. 新しいゲストを追加
`mobile-add-guest.png`
- 最小項目: 氏名 / 役職 / 会社 ＋（任意）好みタグ / NG・苦手タグ / 予算目安。
- 補足「すぐに探せます。詳細は後から追記できます。」CTA「追加してそのまま探す」。

## この UI が体現している設計思想（背景）
- **減点（NG回避）＋加点（センス）の2層**。バッジ＝減点、フィット度＋4軸＋理由＝加点。
- **相手単位フライホイール**: 記録の「良かった点／反省／業務成果」がカルテに蓄積 → 次の探索の理由・スコアに反映（例: 反省「騒がしい」→ 該当店を減点）。
- 詳細は `~/.claude/plans/users-sorasasaki-work-os-02-projects-se-synchronous-popcorn.md`（ブレスト叩き台）。
