# 会食の店選び — 解像度上げリサーチ（2026-07-08）

> 目的: 会食ナビの判定ロジック設計に向けて、「会食の店選び」の暗黙知を外部の実務知見で肉付けする一次調査。
> 位置づけ: `status.md` の**暗黙知9軸（A〜I）**の叩き台を強化し、`docs/spec.md §5.2（判定軸）`・`§5.1（入力フォーム）`・**暗黙知ヒアリングの設問**へ反映するための材料集。
> 手法: sonnet subagent 4本を並列で Web リサーチ（全ソースは末尾）。ルール化候補は `[発火条件] → [店に求める条件/効果]` の型で抽出済み。

## 0. エグゼクティブサマリー（設計に効く5つの発見）

1. **「選んだ店 ＝ 自社の評価」**。店選びの失敗は不手際でなく「センスと本気度」として記憶される。会食は日時調整より先に**店選び・下見が成否を分ける工程**。
2. **予算には税制の裏付けがある**。1人5,000円（会議費/交際費の旧分岐）→ 2024年改正で**1万円**が実質非課税ライン。「1万円前後」の相場感はここが根拠。役職・目的で 6,000〜15,000 円にレンジが割れる。
3. **9軸のうち Places で取れないもの（A格式・B個室の質・C黒子力・I粋）ほど価値が高い**という spec の仮説を、外部知見が裏づけた。特に**個室は「有無」でなく「遮音性・座席形式」の質**が会食の実務判断。
4. **「攻めすぎ」は明確な失敗類型**。過度な高級店・予約困難店・サプライズは、初対面/疲れた相手には逆効果。適正ラインは一貫して**「相手が普段使う店より“ちょっとだけ”上」**。
5. **競合製品NG（G軸）は業界横断の一般フレームだが、明文化された事例はビール系列がほぼ唯一**。自動車・金融・通信の具体タブーは今回のWeb調査では実証ソースが取れず（＝ヒアリングで直接埋めるべき空白）。→ spec の「ビールは第1インスタンス、看板は汎用フレーム」という設計判断と整合。

## 1. 9軸（A〜I）への知見マッピング

各軸に対し、今回のリサーチで**確度が上がった点／新たに追加すべき点／ルール化候補**を整理する。太字は「現行 spec に無く、追加を推奨する要素」。

### A. 格式の釣り合い 
- **確度UP**: 「格下すぎ＝軽視・本気度を疑われる」「格上すぎ＝相手が恐縮・対等でなくなる・次回以降の格を落とせない」の双方向リスクが複数ソースで一致。適正は**「相手が普段使う店よりちょっとだけ良い」**。
- **素人でも掴める格式シグナル**: ①客単価 ②老舗度 ③ミシュラン等の受賞（一つ星＋個室が「安定して機能する」目安）④ジャンル（懐石・割烹＝高格） ⑤立地・空間の特別感（庭園・眺望）⑥**予約時の電話対応の丁寧さ**（当日サービス水準の先行指標）。
- **ルール**:
  - `[相手が部長〜役員 / 重要クロージング] → 客単価1万円以上・老舗 or ミシュラン一つ星クラス・個室完備`
  - `[初対面・関係浅い] → 過度に格式張らない、相手が気負わない範囲`

### B. 個室の"質"（← 最も解像度が上がった軸）
- **有無でなく質**。「個室あり」表記でも実態は**半個室＝声が筒抜け**が頻出。石膏ボード/木の仕切りは音を通す。
- **確認チェックリスト**（下見・電話で聞く）: ①天井まで壁があるか（腰高・欄間の空きはNG）②ドアの有無・厚み（引き戸のみは漏れる）③隣室・通路との距離・往来 ④人数に対する広さ・卓間隔 ⑤内装が格に見合うか。
- **座席形式**: 「**役員に正座はNG**」が鉄則。座敷限定は避け、掘りごたつ（脚を下ろせる＋スーツが皺になりにくく半個室扱いも多い）or 椅子席。年配・体格のある役員は椅子席が確実。
- **ルール**:
  - `[機密性ある商談 / 役員同席] → 完全個室（天井まで壁＋ドア）を必須条件に`
  - `[座敷しか個室が無い] → 掘りごたつ/椅子席の有無を予約前に確認、無ければ除外`
  - `[人事・金額など声量に配慮が要る話題] × [半個室しか空きなし] → 話題を避けるか店を変える`

### C. 会食オペ適性（店の"黒子力"）
- **黒子力の中身**: ①会計を席で見せない/中座会計・伝票非開示 ②料理ペースを商談進行に合わせる ③アレルギー/苦手を厨房まで確実に伝達 ④「いつもの」が効く・名前で呼ぶ ⑤遅刻・急変への安心対応 ⑥タクシー手配・後日請求。
- **事前の見分け方**（データ化不可の軸を"代理指標"で埋める）:
  - **電話対応の質＝店の質**。要望（会食利用/アレルギー/音漏れ/支払い方法）への回答が的確か。曖昧なら当日オペのリスクサイン → **候補から外す判断材料**。
  - **一見の人気店より「勝手を知っている・融通が利く店」**の方が会食の再現性が高い（味だけで選ばない）。
  - 清潔感・スタッフの立ち居振る舞い。
- **ルール**:
  - `[格上の会食] → 中座会計・伝票非開示に対応できる店（予約時に伝えて反応を見る）`
  - `[電話予約時の回答が曖昧] → リスクサインとして減点/除外`
- **示唆（moat 補強）**: この軸は「一度使わないと分からない」ため、**社内記録（会食実績タグ）でしか蓄積できない**という spec の moat 仮説を、外部知見も同じ結論で裏づけている。

### D. 立地・動線
- **駅・相手オフィスからの近さが最優先**。行きづらい立地はそれだけで減点。
- **タクシーの拾いやすさ**（大通り沿い/横付け可）、**雨天動線**（屋根連続・地下直結・傘貸出）、**送りの手配**（タクシーチケットで立替させない、見送りは車が角を曲がる/改札を通るまで）。
- 分かりにくい立地なら**事前に道順をメール**、雑居ビルは**入口・EVホールで出迎え**。
- **ルール**:
  - `[電車来店/終電を気にする年代] → 駅近・分かりやすい動線、二次会も徒歩圏`
  - `[雨予報] → 屋根連続 or 地下直結、傘貸出確認`
  - `[役員が車で来店] → 店前横付け可・近隣P/バレー確認`

### E. 料理・酒 × 相手適合
- **事前ヒアリング定番セット**: ①苦手食材・ジャンル ②アレルギー（命に関わる＝マスト）③少食か ④酒の好み（日本酒/ワイン/焼酎/ビール銘柄/下戸）⑤喫煙 ⑥出身地・趣味（好物推定＋会話ネタ）。
- **聞き方**: ×「食べられないものは？」（子供扱い）→ ○「苦手な食材やアレルギーはございますか？」。直接聞きにくければ「海鮮はお好きですか？」等の選択肢提示。日頃の雑談メモを代用も可。
- **出身地の一手**: 郷土色のある店/一品は会話のきっかけとして機能（＝加点）。
- **下戸配慮**: ノンアル充実・乾杯もソフトドリンク可・無理に勧めない。**過度な特別扱いは逆効果**（当事者の声）。
- **ジャンル別 会食適性**（医師調査の人気順＋実務観点。詳細は §3）:

  | ジャンル | 無難さ | 会話のしやすさ | 実務リスク（汚れ/匂い/取分け） | 向くシーン |
  |---|---|---|---|---|
  | 寿司 | ◎ | ◎(カウンター) | 低 | 顔合わせ・格式全般 |
  | 懐石・会席 | ◎ | ◎(コース進行) | 低 | 顔合わせ・格式全般 |
  | 天ぷら | ○ | ○(職人と対話) | 低 | 顔合わせ〜商談 |
  | フレンチ | ○ | ○ | 低 | 顔合わせ〜御礼 |
  | イタリアン | ○ | ◎ | 低〜中 | 女性同席・カジュアル格式 |
  | 鉄板・ステーキ | ○(華やか) | ◎(シェフと近い) | 低〜中 | 祝い・御礼 |
  | ふぐ・かに | ○(季節/特別) | 普通 | 中(かに=手が汚れる) | 季節会食 |
  | 焼肉 | ✕(人気最下位) | 中(焼き係で分断) | **高**(油はね・匂い・目下が焼く) | 慰労・懇親のみ |
  | 中華(円卓) | 中(円卓マナー要) | ◎(大人数) | 中(取分け・マナー) | 大人数の懇親・祝い |

- **ルール**:
  - `[アレルギー/宗教/健康制約あり] → 予約時に店へ申し送り＋代替コース可否を確認`
  - `[出身地判明] → 郷土色のある店を優先候補（加点）`
  - `[酒に指定銘柄がある企業文化] → 予約時に取扱酒/ビール銘柄を確認`

### F. シーン適合（会の目的）
- 体系的な単一権威記事は無く、断片を統合した傾向（詳細 §3）:

  | シーン | 店に求める性格 | NG例 |
  |---|---|---|
  | 顔合わせ・初対面 | 和食中心・静かな個室・控えめな格 | 攻めた店/騒がしい店 |
  | 御礼 | 相手の好みに寄せた華やかさ・印象に残る一皿 | 顔合わせと同じ無難な和食 |
  | クロージング | 落ち着いた個室・静音・本音を引き出す環境 | 騒がしい店 |
  | 謝罪 | 「普段よりちょっと良い」程度に抑制・清潔・静か | 過度な高級店/高級酒（＝警戒される）|
  | 慰労・懇親 | カジュアル可・居心地優先 | 格式張りすぎ |
  | 祝い | 華やかさ・ライブ感・サプライズ対応 | 地味 |

- **重要な反直感**: **謝罪で高級店・高級酒は逆効果**（「貸しを作って丸め込む気か」と警戒）。誠意は店の格でなく対応の丁寧さで伝える。**トラブル直後の即会食設定も避ける**。

### G. 相手企業・業界タブー
- **競合製品NG**: 「うちは○○グループだから△△ビールしか飲まない」というキーマン（特に年配決裁者）が実在。**企業の資本・系列・グループ相関を事前に調べる**のが基本動作。
- **重要な限界（＝ヒアリングで埋める空白）**: ビール系列**以外**の業界（自動車の系列/金融/通信）の明文化タブーは**今回のWeb調査では実証ソースが取れなかった**。推測断定は危険領域 → spec の「ビール＝第1インスタンス、看板は汎用フレーム」の設計判断は妥当。具体の業界別タブーは**社内ヒアリングで一次採取**すべき。
- **同業競合との"かぶり"**: 同業他社の会食と同じ有名店を使うのはNG（特に歓送迎会シーズンの繁忙期）。
- **三大タブー会話**: 政治・宗教・プロ野球。＋出身校/年齢の決めつけも地雷。会話は**自分3割・相手7割**。
- **ルール**:
  - `[相手が特定メーカー系列/グループ] → 競合酒類・製品を主力にする店はNGフラグ／系列ブランドを扱えば加点`
  - `[相手が同業と競合] → 同業会食とかぶる有名店は避ける`

### H. 予約の堅牢性
- **予約時に伝える情報**: 日時・人数（未確定は「最大◯/最少◯」の幅）／ゲスト・ホスト内訳・男女比／**会食である旨**／食の好み・アレルギー・宗教／飲み物希望（ノンアル含む）／交通手段（社用車台数・タクシー）／手土産の有無・保管／**支払い方法（カード可否・請求書・当日現金かカードか）**。
- **予約困難店は会食に不向きな場合がある**（融通が利かない）。無理に狙うより、幹事が押さえられる店の手数を持つ方が堅牢。
- **ドタキャン/変更耐性**: 減少判明は「分かった瞬間」に連絡。キャンセル料はコース予約で3日前〜前日、席のみで前日〜当日発生が一般的 → **キャンセルポリシーを予約時に確認**。
- **下見が理想**（個室・上座下座位置・トイレ・料理・サービス・支払い方法）。不可なら電話で同等確認。
- **ルール**:
  - `[初めて使う店] → 下見必須、無理なら電話で音漏れ・広さ・座席形式・支払い方法を確認`
  - `[人数流動的] → 幅で予約＋キャンセルポリシー事前確認`
  - `[重要会食] → 話題の予約困難店より、勝手を知っている融通の利く店`

### I. 粋・加点
- **加点になる打ち手**: 予約困難店を押さえた事実そのもの（顔が利く証明）／隠れ家・一見お断り（相手の好みリサーチ前提）／行きつけ・顔なじみ（席の融通・二次会対応）／**季節感・旬**（はしり/旬/名残）／社内ネットワークで相手の好み情報を仕入れる／相手の家族への手土産カスタム／海外ゲストには「日本ならでは」体験。
- **「攻めすぎ」で滑るケース**（＝**減点ルール**として重要）:
  - `[相手が多忙・疲労想定（深夜アポ/連日会食）] → 予約困難店・過度な格式は減点。気疲れしないカジュアル/行きつけを優先`
  - `[初回・関係浅い相手に極端な攻め演出（サプライズ/大仰な予約困難店）] → 滑りリスク大。まず格を合わせた手堅い店`
  - サプライズは「注目が苦手なタイプ」には逆効果 → 延期する勇気も上級者判断。

## 2. 予算相場ベンチマーク（seed タグ・判定に使える数値）

| 予算帯（1人） | 想定シーン | 出典の裏付け |
|---|---|---|
| 6,000〜8,000円 | 気心の知れた相手・関係維持・顔合わせ | 一般会食の目安 7,000〜8,000円（食事2-3千＋酒5千 or 食事5千＋酒3千） |
| 8,000〜12,000円 | 標準的な会食（個室完備が基本ライン） | マイナビ調査で1万〜1.5万が最多(20.3%)、1万以上が3割超 |
| 12,000〜15,000円 | 重要顧客・格式重視・クロージング前後 | 実務ガイドの重要会食帯 |

- **税務の分岐点（相場の裏付け）**: 旧来 1人5,000円が会議費/交際費の分岐。**2024年4月改正で会食飲食費1万円以下は交際費等から除外可**（書類保存が条件）→ 「1万円前後」の相場感の根拠。
- **役員の裁量**: 経営者・役員調査で1回5,000〜10,000円が最多(51.9%)。役員の73.8%が交際費を「使える」、55.3%は「上限なく使える」。ただし社内上限の設定は推奨。
- **安すぎ/高すぎのリスク**: 安すぎ＝軽視・本気度を疑われる／高すぎ＝恐縮・下心を疑われる・経理チェックで問題視。

## 3. シーン別・ジャンル別の詳細（フォーム設計の根拠）

（§1 F・E のマトリクスの補足。**入力フォームに「会の目的」「相手の役職」「相手の嗜好」を必須化する根拠**になる。）

- **顔合わせ**: 好み不明の段階では**和食が最も外しにくい**。静かな個室・控えめな格。
- **御礼**: 相手の好みジャンルに寄せ、**印象に残る一皿・体験**を。顔合わせと同じ無難和食は失敗。
- **クロージング**: 「会食＝情報交換と相互理解の場」。**個室・静音**で本音を引き出せる環境が信頼構築に直結。
- **謝罪**: 過度な高級店・高級酒は警戒される。**清潔・静か・誠実な接客**で。トラブル直後の即設定は避ける。
- **慰労・懇親**: 高級・格式は不要。**居心地優先**。焼肉等カジュアルもここなら可。
- **祝い**: 鉄板焼き等**ライブ感・サプライズ対応**の個室。

**焼肉が会食人気最下位の理由**（判定で減点する根拠）: ①匂いが服に付く ②油はねで汚れる ③**目下が焼き役になり会食に参加できない**。フォーマル会食には不向き、慰労・懇親限定。

**中華円卓のマナーリスク**: 回し方（時計回り・上座から）、全員取り分け完了まで食べ始めない等、知らないと恥をかく → 大人数懇親向きだが事前レクチャー推奨。

## 4. 食のNG 深刻度ランク（NGフラグの重み付け）

| 深刻度 | 対象 | 回避策 | 判定での扱い |
|---|---|---|---|
| **重大** | アレルギー・宗教禁忌（ハラル/豚・牛不可/ベジ/コーシャ） | 予約時に食材・調理法を確認、代替コース相談まで必須 | 確実な時は**NGフラグ**（除外はせず理由付きで残す） |
| **中度** | 健康制約（痛風=プリン体・ビール、糖尿、肝臓） | 無理に勧めず、コース以外に軽め/単品がある店を選ぶ | 減点＋理由文 |
| **軽度** | 単純な苦手（パクチー等） | アラカルトで調整可能な店の方がリスク低 | 減点（微） |

- **外国人ゲスト**は日本人以上に宗教・食習慣・アレルギー制限が多い。インド人（ヒンドゥー=牛不可）・ユダヤ人（コーシャ）・ムスリム（豚・酒不可、右手食事）は入念な事前確認。国交省に公的対応マニュアルあり。

## 5. 「新人がやらかす典型」カタログ（テスト/デモの反例に使える）

spec §5.1 の「NGも理由付きで残す」UX で、**新人教育の反例**として提示できる典型集:

1. ネット口コミスコアだけで店を決める（実地確認しない）→ 観光客向け/会食不向きだった。
2. 「個室あり」表記だけで安心 → 実態は半個室で商談が隣卓に丸聞こえ。
3. 座敷限定の店を予約 → 正座に慣れない役員を困らせた。
4. 相手の役職を無視し自社の格を合わせない/店の格が不釣り合い（役員にチェーン個室・食べログ3.5）。
5. アレルギー・苦手のヒアリングを省略 → 当日食べられない皿が出る。
6. 雰囲気重視で路地裏の隠れ家 → タクシーが拾えず相手を歩かせた/終電で駅まで走る。
7. 会計を相手の目の前で処理 → 気を遣わせた/小銭でもたつく。
8. 予約困難店を無理に押さえた → 勝手が分からず当日バタつく。
9. キャンセルポリシー未確認で人数確定 → 直前変更で高額キャンセル料。
10. 謝罪で高級店・高級酒 → 逆に警戒される。
11. 会食中に自分がしゃべりすぎる/酒の勢いで本音を強引に聞き出す。
12. 手土産を床に置く/名前を呼び間違える/上座下座を誤る。
13. 翌日のお礼メールを忘れる。

## 6. ヒアリング設問への反映提案（status.md 次アクションへの入力）

今回の調査で**「Webで埋まった軸」と「社内ヒアリングでしか埋まらない軸」が明確に分離**できた。ヒアリングは後者に集中させると効率が良い。

### Web調査で概ね埋まった（ヒアリングは検証・重み付けだけでよい）
- **D 立地・動線** / **H 予約堅牢性** / **E アレルギー・宗教・健康の禁忌** / **F シーン別の大枠** / **予算相場**。

### 社内ヒアリングで一次採取すべき空白（＝設問を厚くする）
- **G 業界別タブーの具体**: ビール系列以外に、御社の取引先（自動車/金融/通信/メーカー等）で「この製品・系列の店はNG/歓迎」という具体例はあるか？ ← Webでは取れなかった最大の空白。
- **A 格式ランクの付け方**: 「この店はS/A/B」をベテランはどの基準で判断しているか（客単価だけでない暗黙のシグナル）。
- **C "会食慣れした店"の見分け方**: 電話対応・下見のどこを見て「黒子力がある」と判断するか。過去に「黒子力で救われた/失敗した」実例。
- **I 粋の当たり/滑りの分水嶺**: どういう相手・シーンで「攻め」が効き、どこから滑るか。実際に滑った事例。
- **重み**: 9軸のうち、実際に「これを外すと一発アウト」なのはどれか（NGフラグの強度ランク付け）。
